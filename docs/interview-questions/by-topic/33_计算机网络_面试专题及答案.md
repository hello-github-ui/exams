# 33_计算机网络_面试专题及答案

1. TCP建⽴立连接的过程。
三次握⼿手：
1. 第⼀一次握⼿手(客户端发送syn包到服务器器端)：客户端发送syn包到服务
器器端，进⼊入syn_send状态，等待服务器器端的确认；
2. 第⼆二次握⼿手(服务器器返回syn+ack包给客户端)：服务器器端收到客户端
的syn包，发送syn+ack包给客户端，进⼊入syn_recv状态；
3. 第三次握⼿手(客服端返回ack包给服务端)：客户端收到服务器器端的
syn+ack包，发送个ack包到服务器器端，⾄至此，客户端与服务器器端进⼊入
established状态；
4. 握⼿手过程中传送的包不不包含任何数据，连接建⽴立后才会开始传送数
据，理理想状态下，TCP连接⼀一旦建⽴立，在通信双⽅方的任何⼀一⽅方主动关闭
连接前，TCP连接都会⼀一直保持下去。
2. TCP断开连接的过程。
四次挥⼿手：
1. 第⼀一次挥⼿手：主动关闭⽅方发送fin包到被动关闭⽅方，告诉被动关闭⽅方我
不不会再给你发送数据了了；
2. 第⼆二次挥⼿手：被动关闭⽅方收到syn包，发送ack给对⽅方，确认序号为收
到序号+1；
3. 第三次挥⼿手：被动关闭⽅方也也发送fin包给主动关闭⽅方，告诉对⽅方我也
不不会给你发送数据了了；
4. 第四次挥⼿手：主动关闭⽅方收到syn包，发送ack给对⽅方，⾄至此，完成四
次挥⼿手；

4. HTTP协议的交互流程。 HTTP和HTTPS的差异， SSL的交互流程？
a. Http协议
1、建⽴立TCP连接；
2、发送HTTP请求；
3、服务器器处理理请求；
4、返回响应结果；
5、关闭TCP连接；
b. http三次握⼿手：
 
i.         第⼀一次握⼿手：客户端发送syn包(syn=j)到服务器器，并进⼊入
SYN_SEND状态，等待服务器器确认；
ii.         第⼆二次握⼿手：服务器器收到syn包，必须确认客户的
SYN（ack=j+1），同时⾃自⼰己也发送⼀一个SYN包（syn=k），即
SYN+ACK包，此时服务器器进⼊入SYN_RECV状态；
iii.         第三次握⼿手：客户端收到服务器器的SYN＋ACK包，向服务器器
发送确认包ACK(ack=k+1)，此包发送完毕，客户端和服务器器进⼊入
ESTABLISHED状态，完成三次握⼿手。

HTTPS协议就是基于SSL的HTTP协议
HTTPS使⽤用与HTTP不不同的端⼝口（HTTPS80 ， HTTPSS443）
提供了了身份验证与加密通信⽅方法，被⼴广泛⽤用于互联⽹网上安全敏敏感的
通信。
1、客户端请求SSL连接，并将⾃自⼰己⽀支持的加密规则发给⽹网站。
2、服务器器端将⾃自⼰己的身份信息以证书形式发回给客户端。证书⾥里里
⾯面包含了了⽹网站地址，加密公钥，以及证书的颁发机构。
3、获得证书后，客户要做以下⼯工作
验证证书合法性
如果证书受信任，客户端会⽣生成⼀一串串随机数的密码，并⽤用证
书提供的公钥进⾏行行加密。
将加密好的随机数发给服务器器。
4、获得到客户端发的加密了了的随机数之后，服务器器⽤用⾃自⼰己的私钥
进⾏行行解密，得到这个随机数，把这个随机数作为对称加密的密钥。
（利利⽤用⾮非对称加密传输对称加密的密钥）
5、之后服务器器与客户之间就可以⽤用随机数对各⾃自的信息进⾏行行加
密，解密。

注意的是：证书是⼀一个公钥，这个公钥是进⾏行行加密⽤用的。⽽而私钥是
进⾏行行解密⽤用的。公钥任何都知道，私钥只有⾃自⼰己知道。这是⾮非对称
加密。
⽽而对称加密就是钥匙只有⼀一把，我们都知道。
之所以⽤用到对称加密，是因为对称加密的速度更更快。⽽而⾮非对称加密
的可靠性更更⾼高。
 
客户端请求--服务端发送证书（公钥）--客户端验证证书，并⽣生成
随机数，通过公钥加密后发送给服务端--服务端⽤用私钥解密出随机
数--对称加密传输数据。
d. HTTP与HTTPS的区别
1、HTTPS协议需要申请证书。
2、HTTP是明⽂文传输；HTTPS使⽤用的是具有安全性的SSL加密传输
协议
3、HTTP端⼝口是80；HTTPS端⼝口号是443
4、HTTP连接简单⽆无状态；HTTPS由SSL+HTTP协议构件的可进⾏行行
加密传输、身份验证的⽹网络协议。

8. 交换机与路路由器器的区别？
